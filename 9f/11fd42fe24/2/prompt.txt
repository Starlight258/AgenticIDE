왜 이거: 단순 readme 아니라 "스타트업에서 몇 년 운영한 후 정리한" 실전 가이드. AGENTS.md 파일도 함께 제공 — AI agent용 머신 리더블 포맷이라 Cursor/Claude Code에서 바로 룰로 쓸 수 있음. DH가 AGENTS.md 강조하는 거 생각하면 더 가치 있음.
핵심 내용:

프로젝트 구조 — Netflix Dispatch 방식. 도메인 기반 분리 (src/auth/, src/posts/ 등). 각 모듈에 router.py, schemas.py, models.py, service.py, dependencies.py 패턴
async vs sync 라우트 — time.sleep을 async 라우트에서 쓰는 게 왜 치명적인지, threadpool 동작 원리
Pydantic 광범위 사용 — Field(min_length=..., max_length=..., pattern=...), EmailStr, AnyUrl 활용
Custom BaseModel — 글로벌 datetime serializer 등 표준 형식 강제
Dependencies — DI를 단순 의존성 주입이 아니라 request validation 도구로 쓰는 법. dependency chaining + caching
BackgroundTasks vs Celery — 어떤 작업을 어디 두는지 명확한 기준 표
DB 네이밍 컨벤션 — snake_case, _at/_date suffix, 모듈 prefix
SQL-first, Pydantic-second — 복잡한 join은 DB에서, 응답 nested object도 DB에서 JSON aggregate
httpx + ASGITransport — 비동기 테스트 클라이언트 셋업
dependency_overrides — 테스트에서 monkeypatch 대신 의존성 교체
