@case-01-session-second/ 에서 이 실수를 했는데 어떤 문제야? 
다만 큰 감점이 있습니다. second는 case-01-session-second/src/llm.py:67에서 AGENTS.md 내용을 LLM 프롬프트에
    넣지 않고, case-01-session-second/src/guardrails.py:9도 파일을 읽지 않습니다. 과제의 “Guardrails MUST
    consult brand's AGENTS.md”를 엄격히 보면 P1입니다. 이게 무슨 말이야?


• 말 그대로 second 구현은 efood의 AGENTS.md를 실제 실행 경로에서 참고하지 않고, 규칙을 코드에 하드코딩했다는
  뜻입니다.

  과제 요구사항은:

  POST /sessions/{id}/patches/{patchId}/check

  - patch에 guardrail을 실행한다
  - Guardrails MUST consult the brand's AGENTS.md

  즉 평가자가 기대할 수 있는 흐름은 대략 이겁니다.

  # check 실행 시점
  agents_text = read("efood/AGENTS.md")
  rules = derive_or_confirm_rules_from(agents_text)
  checks = run_guardrails(patch, rules)

  그런데 case-01-session-second는 case-01-session-second/src/guardrails.py:9에서 바로 이렇게 돼 있습니다.

  def run_checks(diff: str) -> list[GuardrailCheck]:
      added = _added_lines(diff)
      return [
          _check("R1", "WARN", _has_relative_import, added),
          _check("R2", "BLOCK", _has_shell_execution, added),
          _check("R3", "WARN", _has_public_function_without_docstring, added),
          _check("R4", "BLOCK", _has_print_call, added),
          _check("R5", "BLOCK", _has_requests_call, added),
      ]

  여기에는 efood/AGENTS.md를 읽는 코드가 없습니다. case-01-session-second/src/llm.py:67의 프롬프트도:

  f"Create an implementation plan for a {brand} AI coding session.\n"

  정도라서 AGENTS.md 내용을 LLM에게 주지도 않습니다.

  반대로 backend는 완벽하진 않아도 case-01-session-backend/src/llm.py:12에서:

  def _read_agents(brand: str) -> str:
      return Path(f"{brand}/AGENTS.md").read_text()

  이렇게 브랜드별 AGENTS.md를 읽고, system prompt에 넣습니다.

  내가 “P1”이라고 한 이유는, second의 guardrail 결과가 지금은 우연히 AGENTS.md와 맞지만, 파일을 바꿔도 동작이
  안 바뀝니다. 그러면 “브랜드 manifesto를 consult한다”는 제품 컨셉이 약해집니다. 다만 90분 과제 기준으로는
  “R1-R5를 정확히 구현했느냐”를 더 크게 볼 수도 있어서, 이걸 P2로 낮게 보는 평가자도 있을 수 있습니다.
