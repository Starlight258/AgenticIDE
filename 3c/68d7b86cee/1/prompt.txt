› 테스트면 원래 mock으로 해야되는거아니야? 이거 빼먹은거지? agent가?


• 네, 이건 agent가 빼먹은 게 맞습니다.

  테스트, 특히 pytest로 도는 E2E/통합 테스트는 기본적으로:

  - 외부 API 호출하지 않아야 하고
  - API 키 유무에 따라 결과가 달라지면 안 되고
  - LLM 응답 개수/내용 같은 비결정성에 의존하면 안 됩니다.

  case-01-session-second는 mock fallback 자체는 만들었지만, 테스트에서 mock 모드를 강제하지 않았습니다. 그래
  서 로컬/CI 환경에 ANTHROPIC_API_KEY가 있으면 실제 LLM을 호출합니다. 이건 테스트 설계 실수입니다.

  제대로 하려면 보통 둘 중 하나를 해야 합니다.

  # tests/conftest.py
  import os

  os.environ["ANTHROPIC_API_KEY"] = ""

  또는 더 깔끔하게는 llm.create_plan, llm.create_patch를 monkeypatch해서 테스트 중에는 항상 고정 응답을 쓰게
  해야 합니다.

  그리고 테스트도 len(plan) == 1처럼 mock 구현 세부사항에 너무 묶기보다, 최소 계약을 검증하는 게 낫습니다.

  assert len(plan) >= 1
  assert "patches" not in plan[0]
